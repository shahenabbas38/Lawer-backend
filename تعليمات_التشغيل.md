# تشغيل الباك اند (Laravel) لوكال

## هل تحتاج XAMPP؟

- **نعم** إذا استخدمت **MySQL** في قاعدة البيانات (إعدادات `DB_CONNECTION=mysql` في `.env`).
- **لا** إذا استخدمت **SQLite** (إعدادات `DB_CONNECTION=sqlite` في `.env`) — لا تحتاج تشغيل MySQL ولا Apache.

في الحالتين **لا تحتاج تشغيل Apache** من XAMPP؛ Laravel يشتغل بأمر `php artisan serve`.

---

## الطريقة 1: مع XAMPP (MySQL)

### 1) تشغيل XAMPP
- افتح **XAMPP Control Panel**.
- شغّل **MySQL** فقط (اضغط Start بجانب MySQL).
- لا يلزم تشغيل Apache.

### 2) إعداد ملف `.env`
- في مجلد المشروع `lawyer_backend` تأكد من وجود ملف `.env` (إن لم يكن، انسخ من `.env.example`).
- تأكد أن إعدادات قاعدة البيانات كالتالي (قيم XAMPP الافتراضية):

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=lawyer_db
DB_USERNAME=root
DB_PASSWORD=
```

- أنشئ قاعدة البيانات `lawyer_db` من **phpMyAdmin** (من XAMPP: http://localhost/phpmyadmin) أو من MySQL:

```sql
CREATE DATABASE lawyer_db;
```

### 3) أوامر التيرمنال في مجلد الباك اند

```powershell
cd C:\Users\hp\lawyer_backend
```

أول مرة فقط:

```powershell
copy .env.example .env
php artisan key:generate
php artisan migrate
php artisan storage:link
```

ثم في كل مرة تريد تشغيل الباك اند:

```powershell
php artisan serve --host=0.0.0.0
```

اترك التيرمنال مفتوحًا. الباك اند يعمل على: **http://127.0.0.1:8000**  
والتلفون يوصله عبر: **http://192.168.10.123:8000** (نفس الـ IP اللي في التطبيق).

---

## الطريقة 2: بدون XAMPP (SQLite)

إذا كان في `.env`:

```env
DB_CONNECTION=sqlite
```

(وباقي سطور MySQL معطّلة أو محذوفة)

لا تحتاج تشغيل أي شيء من XAMPP.

أول مرة فقط:

```powershell
cd C:\Users\hp\lawyer_backend
copy .env.example .env
php artisan key:generate
# إنشاء ملف SQLite الفارغ إن لم يكن موجوداً
echo. > database\database.sqlite
php artisan migrate
php artisan storage:link
```

ثم في كل مرة:

```powershell
php artisan serve --host=0.0.0.0
```

## ملخص سريع (للتجربة مع الموبايل)

| الخطوة | الأمر / الإجراء |
|--------|------------------|
| 1 | تشغيل **MySQL** من XAMPP فقط (إن كنت تستخدم MySQL) |
| 2 | `cd C:\Users\hp\lawyer_backend` |
| 3 | أول مرة: `php artisan migrate` و `php artisan storage:link` |
| 4 | `php artisan serve --host=0.0.0.0` |
| 5 | التطبيق على الموبايل يربط على `http://192.168.10.123:8000` |

**مهم:** الموبايل والكمبيوتر على **نفس شبكة الـ Wi‑Fi**.

---

## إذا 192.168.10.123:8000 ما يعطي استجابة

1. **شغّل الباك اند بهذا الأمر (مهم):**
   ```powershell
   php artisan serve --host=0.0.0.0
   ```
   بدون `--host=0.0.0.0` السيرفر يسمع فقط من جهازك (127.0.0.1) وما يقبل اتصال من الموبايل.

2. **تأكد من الـ IP:** في PowerShell على الكمبيوتر:
   ```powershell
   ipconfig
   ```
   اختر **IPv4 Address** لاتصال الـ **Wi-Fi** (مثلاً 192.168.10.123). إذا مختلف غيّره في تطبيق Flutter (`api_base_url_io.dart`) وفي الرابط اللي تجربه.

3. **جدار النار (Firewall):** قد يمنع ويندوز البورت 8000. جرّب إما:
   - فتح بورت 8000 لشبكتك الخاصة، أو
   - عند ظهور نافذة "Windows Security" لـ "PHP" أو "Private network" اختر **Allow access**.

4. **تجربة من الكمبيوتر نفسه:** افتح المتصفح على الكمبيوتر واكتب:
   ```
   http://192.168.10.123:8000
   ```
   إذا من الكمبيوتر يفتح ومن الموبايل لا = غالباً شبكة أو جدار نار. إذا من الاثنين لا = تأكد أن `php artisan serve --host=0.0.0.0` شغال وأن الـ IP صحيح.
